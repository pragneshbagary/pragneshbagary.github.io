About

A bit about me

I'm an engineer based in Hyderabad with an M.S. in Computer Engineering from San Jose State University. I work across two threads: clean, well-tested Go systems - libraries you can actually compose without coupling - and LLM / agent tooling built on local-first primitives.

Right now I'm developing go-thin, an open-source suite of interface-driven Go libraries, and memex, an MCP server that gives coding agents persistent per-project memory. Alongside that I reimplement reinforcement-learning research from first principles and contribute patches to projects like charmbracelet/crush and lima-vm/lima.

I care about correctness, small API surfaces, and results that survive a second look.
